0) 기술 스택/구성

프론트엔드: React + TypeScript (Vite)

UI: TailwindCSS (또는 shadcn/ui 사용 가능)

백엔드: Node/Express 또는 Python/FastAPI 중 하나 (Replit에서 실행 가능한 방식)

데이터 저장: 우선은 localStorage 또는 SQLite(선택)로 구현해도 됨

코드는 실행 가능해야 하고, README에 실행 방법을 포함해줘.

1) 핵심 화면 요구사항 (필수)

음식 메뉴 후보 여러개를 띄워주는 화면이 있어야 해.

“오늘의 추천 3개” + “추가 후보 10개” 같은 형태로 보여줘.

각 메뉴 카드에는 최소한 메뉴명, 태그(카테고리), 추천 이유(한 줄)를 표시해줘.

개인화 옵션이 다양해야 해.

사용자가 선택할 수 있는 옵션(예시):

선호 카테고리(한/중/일/양/분식 등)

밥/면/빵/샐러드

국물/비국물

단백질(돼지/소/닭/해산물/채식)

매운 정도(0~3)

무거움 정도(가벼움~묵직함)

예산(저가/중가/고가)

알레르기/제외 재료(선택)

옵션은 “필터”가 아니라 “선호”로 반영되도록 해줘(완전 제외가 아니라 가중치로).

깔끔한 UI/UX를 구성해줘.

카드 기반 레이아웃, 여백 충분히, 모바일에서도 보기 좋게.

사용 흐름:
① 최근 식사 기록 입력 → ② 개인화 옵션 설정 → ③ 추천 보기 → ④ 선택/거절 피드백 → ⑤ 다음 추천 갱신

사용자의 취향 분석을 통해 추천해주는 메뉴를 잘 구성해줘.

최근 7일 식사 기록을 분석해서 “반복 방지(다양성)”를 강하게 적용해줘.

사용자가 선택/거절한 기록을 반영해 선호 가중치를 업데이트해줘.

추천 이유는 반드시 사용자의 기록/옵션/피드백과 연결되어야 해(설명 가능한 추천).

2) 기능 상세 요구사항
A. 최근 7일 식사 기록 입력/관리

입력 필드: 날짜/시간, meal_type(아침/점심/저녁/간식), menu_text(메뉴명)

최근 7일 기록 리스트(추가/삭제) 화면 제공

입력된 menu_text는 정규화 과정을 거쳐 canonical_menu로 매핑되면 좋음(간단한 사전 기반)

B. 메뉴 후보 DB(내장)

최소 200개 이상의 메뉴 후보 리스트를 앱에 내장해줘.

각 메뉴는 다음 정보를 가져야 함:

canonical_menu

tags: cuisine, base, soup, protein, spicy_level, heavy_level, price_range

synonyms(선택)

한국 메뉴만 아니어도 되지만, 다양한 메뉴를 포함해줘(한/중/일/양/분식+글로벌).

C. 추천 로직(설명 가능한 점수화)

추천 점수는 아래 구성 요소로 계산해줘:

preference_score: 개인화 옵션(선호)와의 일치도

diversity_bonus: 최근 7일 기록과 겹치지 않는 정도(메뉴/카테고리)

repetition_penalty: 최근 7일에 등장한 메뉴/유사 카테고리는 큰 페널티

feedback_weight: 과거 선택/거절에 따른 가중치 업데이트

의사코드(또는 실제 코드 주석)로 점수식을 명확히 설명해줘.

출력: 추천 3개 + 후보 10개, 그리고 각 추천 카드에 “왜 추천했는지” 한 줄 생성

D. 피드백(선택/거절/스킵)

추천 카드마다 버튼 제공: “선택”, “거절”, “나중에”

선택/거절 시 사용자의 선호 가중치가 업데이트되어야 함(단순 +1/-1로 시작해도 됨)

추천 결과가 즉시 갱신되도록 해줘.

3) 평가/데모 모드

“데모 데이터 자동 채우기” 버튼을 만들어서 최근 7일 식사 기록이 자동으로 들어가게 해줘.

추천이 어떻게 바뀌는지 확인할 수 있도록 “선호 가중치/최근 카테고리 분포”를 간단히 보여주는 섹션(작은 카드)도 추가해줘.

4) 산출물

실행 가능한 코드

화면 3개 이상(기록 입력/추천 화면/설정 또는 대시보드)

README: 실행 방법, 기능 설명, 추천 로직 요약

코드 주석: 추천 점수 계산과 업데이트 로직은 이해 가능하게 작성

5) UX 문구(필수)

추천 카드에는 아래처럼 구체적인 추천 이유를 반드시 포함해줘(예시):

“최근 7일 동안 면 요리가 많아 오늘은 밥 메뉴를 추천했어요.”

“어제 국물이었으니 오늘은 비국물 위주로 골랐어요.”

“최근 ‘매운맛’ 선택이 많아서 매운 단계 2 메뉴를 우선 추천했어요.”

이 요구사항을 만족하는 PoC를 만들어줘. UI는 미니멀하고 깔끔하게, 기능은 실제로 동작하도록 구현해줘.
